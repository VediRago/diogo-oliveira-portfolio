# Development Diary

This folder records the development of the Erit worldbuilding branch as it evolved from Fragments.

The purpose of the diary is to preserve the design trail behind the world, methods, and prototypes shown throughout the portfolio.

The entries are not intended to function as canon reference files or encyclopedia articles. They document discoveries, corrections, experiments, and design decisions as they occurred.

## Design Principle

The diary follows a simple rule:

```text
experience -> emotion -> reason -> behavior
```

When needed, the chain expands into:

```text
experience -> emotion -> reason -> behavior -> atmosphere -> history
```

Or, for game-facing systems:

```text
experience -> emotion -> reason -> behavior -> atmosphere -> quest logic
```

The goal is to document how ideas developed, not simply what was created.
